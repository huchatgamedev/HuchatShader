# Shader Learning Guide

Unity 6 + URP 環境でのシェーダー学習を体系的に支援するスキル。
CLAUDE.mdで定義された5つのPhaseに基づき、進捗管理・課題提案・理解度確認を行う。

## 学習カリキュラム（Phase一覧）

### Phase 1: 基礎
**目標**: ShaderLabの構文を理解し、簡単なUnlitシェーダーを書けるようになる。

| # | トピック | 成果物 | 確認項目 |
|---|---------|--------|---------|
| 1-1 | ShaderLab構文 | 最小のUnlit Shader | Properties, SubShader, Pass構造を説明できる |
| 1-2 | 色の表示 | 単色Shader | `_BaseColor` のみで色を出力できる |
| 1-3 | テクスチャ表示 | テクスチャShader | TEXTURE2D/SAMPLER宣言、SAMPLE_TEXTURE2Dが使える |
| 1-4 | 座標空間の変換 | 座標ビジュアライズShader | OS→WS→VS→HCSの流れを図示できる |
| 1-5 | UV操作 | UVスクロール/タイリングShader | TRANSFORM_TEX, _Timeを使ったアニメーション |
| 1-6 | Shader Graph比較 | 同等のShader Graph | コードとノードの対応関係を説明できる |

### Phase 2: ライティング
**目標**: 基本的なライティングモデルを手書きで実装できる。

| # | トピック | 成果物 | 確認項目 |
|---|---------|--------|---------|
| 2-1 | Lambert拡散反射 | Diffuse Shader | N·Lの意味を説明できる |
| 2-2 | Blinn-Phong鏡面反射 | Specular Shader | ハーフベクトルの意味を説明できる |
| 2-3 | 環境光 | Ambient追加 | 影の部分が真っ黒にならない理由を説明できる |
| 2-4 | 法線マッピング | Normal Map Shader | Tangent Spaceの必要性を説明できる |
| 2-5 | 影の受け取り | Shadow付きLit Shader | Shadow Coordの仕組みを説明できる |
| 2-6 | 複数ライト | Multi-Light Shader | Additional Lightsの取得方法を説明できる |
| 2-7 | ShadowCaster Pass | 影を落とすShader | ShadowCaster Passの役割を説明できる |

### Phase 3: 応用表現
**目標**: 表現技法を組み合わせてスタイライズドな見た目を作れる。

| # | トピック | 成果物 | 確認項目 |
|---|---------|--------|---------|
| 3-1 | トゥーンシェーディング | Toon Shader | ランプテクスチャ/Step関数の使い分け |
| 3-2 | アウトライン | Outline Shader | 頂点押し出し法の原理を説明できる |
| 3-3 | Rim Light | Fresnel Shader | フレネル効果の数学を説明できる |
| 3-4 | ディゾルブ | Dissolve Shader | ノイズテクスチャとclipの関係を説明できる |
| 3-5 | 透過・半透明 | Transparent Shader | Blend命令とRender Queueを説明できる |
| 3-6 | Alpha Test | Cutout Shader | clip/discardの使い方を説明できる |
| 3-7 | ステンシルバッファ | Stencil Shader | ステンシルテストの仕組みを説明できる |

### Phase 4: 高度な技術
**目標**: エンジンレベルの機能を活用した高度なシェーダーが書ける。

| # | トピック | 成果物 | 確認項目 |
|---|---------|--------|---------|
| 4-1 | Custom Renderer Feature | ポストプロセスShader | Renderer Featureの仕組みを説明できる |
| 4-2 | Compute Shader | 基本Compute | スレッドグループの概念を説明できる |
| 4-3 | GPU Instancing | Instanced Shader | インスタンシングの仕組みを説明できる |
| 4-4 | モバイル最適化 | 最適化済みShader | プロファイリング結果を読めて改善できる |

### Phase 5: 統合・実践
**目標**: 実プロジェクトレベルのシェーダーを設計・実装できる。

| # | トピック | 成果物 | 確認項目 |
|---|---------|--------|---------|
| 5-1 | キャラクターシェーダー | 肌・髪・衣服Shader | 素材ごとの表現を使い分けられる |

| 5-3 | VFXシェーダー | パーティクル・歪みShader | 視覚効果を自在に作れる |
| 5-4 | パフォーマンス最適化 | 最適化レポート | ボトルネック特定と改善ができる |

## 進捗確認の方法

### 現状の把握

ユーザーの進捗を把握するために以下を確認する:

1. **Assets/HuchatShader/Shaders/** 以下のファイルを一覧
2. **Assets/HuchatShader/Docs/LearningLog.md** の内容を確認（あれば）
3. 直近の会話から学習段階を推測

### 進捗の記録

学習完了時に `Assets/HuchatShader/Docs/LearningLog.md` に記録を提案する:

```markdown
## Phase 1: 基礎

### 1-1: ShaderLab構文 ✅
- 完了日: 2024-XX-XX
- 成果物: Shaders/Unlit/SimpleColor.shader
- メモ: Propertiesの型指定を理解した

### 1-2: テクスチャ表示 🔄（進行中）
- 開始日: 2024-XX-XX
- つまずき: CBUFFER内にテクスチャを入れてSRP Batcher非互換になった
```

## 練習課題の設計方針

### 課題のレベル

各トピックで3段階の課題を用意する:

| レベル | 名前 | 内容 |
|--------|------|------|
| ★☆☆ | 写経 | テンプレートをそのまま入力し、動作を確認する |
| ★★☆ | 改造 | 既存シェーダーにパラメータや機能を追加する |
| ★★★ | 自作 | ゼロから要件を満たすシェーダーを書く |

### 課題テンプレート

```
## 課題: [課題名]
**Phase**: X-X
**レベル**: ★★☆
**前提知識**: [必要なトピック]

### 要件
- [ ] 要件1
- [ ] 要件2
- [ ] 要件3

### ヒント
1. ヒント1（折りたたみにする）
2. ヒント2

### 確認ポイント
- SRP Batcher互換であること（Frame Debuggerで確認）
- [特定の見た目になること]
```

## Phase 1 の練習課題例

### 課題1-1: 単色Unlit Shader（★☆☆ 写経）
**要件**:
- `_BaseColor` プロパティで色を変更できるUnlit Shadeを作成
- SRP Batcher互換であること
- `Shader "Custom/Unlit/SimpleColor"` として作成

### 課題1-2: テクスチャ付きUnlit Shader（★☆☆ 写経）
**要件**:
- テクスチャを表示するUnlit Shader
- Tiling と Offset がインスペクターから変更可能
- テクスチャに色を掛け合わせられる

### 課題1-3: UVスクロール Shader（★★☆ 改造）
**要件**:
- テクスチャが時間と共にスクロールする
- `_ScrollSpeedX` と `_ScrollSpeedY` で速度を制御可能
- 元のテクスチャのTiling/Offsetも維持する

### 課題1-4: 座標ビジュアライズ Shader（★★☆ 改造）
**要件**:
- キーワードで表示モードを切り替えられる（法線/UV/ワールド座標）
- 各値を色として出力して視覚的に確認できる
- `shader_feature` を使ってバリアントを管理する

### 課題1-5: チェッカーボード Shader（★★★ 自作）
**要件**:
- UV座標を使ってプロシージャルなチェッカーボードパターンを生成
- マスの数、色A、色Bをプロパティで制御可能
- テクスチャ不要で完全にコードで生成する

## 理解度確認の質問テンプレート

各トピック完了時に、以下の形式で理解度を確認する:

### 知識確認（口頭で回答）
1. [概念の意味を問う質問]
2. [なぜその手法を使うのか理由を問う質問]
3. [コードの特定部分が何をしているか問う質問]

### 実践確認（コードで回答）
1. [既存シェーダーに小さな変更を加える課題]
2. [バグのあるシェーダーを修正する課題]

## 次のステップの提案ルール

1. ユーザーのAssetsフォルダ内のシェーダーファイルを確認する
2. 完了しているトピックを判断する
3. 未完了の最初のトピックを「次のステップ」として提案する
4. 前のトピックの復習が必要そうなら、復習課題を先に出す
5. ユーザーが特定のトピックに興味を示したら、Phase順に関わらず柔軟に対応する（ただし前提知識が足りない場合は補足する）
