# Shader Concept Explainer

Unity 6 + URP 環境でのシェーダー学習における概念・理論の解説スキル。
GPUで何が起きているかを概念的に説明し、数値例や図解を交えて直感的な理解を促す。

## 解説の基本方針

### 説明の構成

概念を説明する際は以下の順序で構成する:

1. **一言で言うと** — 最初に結論を1文で述べる
2. **なぜ必要か** — この概念が存在する理由・解決する問題
3. **仕組み** — 具体的な動作原理。数値例や図を使う
4. **コードとの対応** — HLSLコードおよびShader Graphノードでどう書くか
5. **よくある落とし穴** — 初学者が間違えやすいポイント
6. **パフォーマンスへの影響** — 該当する場合のみ

### 図解のルール

- ASCII図やMermaidダイアグラムを積極的に使う
- 座標変換は必ず数値例を添える
- ベクトル演算は2D/3Dの直感的な図で示す

### 学習Phaseとの連携

CLAUDE.mdで定義された5つのPhaseに基づき、現在の学習段階に合わせた深さで説明する:

- **Phase 1-2の概念**: 詳細に解説。図解・数値例を豊富に
- **Phase 3-4の概念が質問された場合**: 概要を説明し、「Phase Xで詳しく学ぶ」と案内
- **Phase 5の概念**: 概要のみ触れ、前提知識を先に学ぶよう促す

## トピック別ガイダンス

各トピックで解説する際のポイントと必須事項。

### 座標空間と変換

**必ず含めるべき内容**:
- 5つの座標空間（Object → World → View → Clip → NDC/Screen）
- 各変換の直感的な意味
- 変換行列の概念
- URPでの対応関数

**数値例テンプレート**:
```
Object Space: (1, 0, 0)  ← モデルのローカル座標
    ↓ Model行列（位置・回転・スケールを適用）
World Space:  (5, 2, 3)  ← ワーム内の絶対位置
    ↓ View行列（カメラ基準に変換）
View Space:   (2, 1, -5) ← カメラから見た相対位置（Z=奥行き）
    ↓ Projection行列（遠近法 or 平行投影）
Clip Space:   (0.4, 0.2, 0.8, 5.0)  ← 同次座標
    ↓ 透視除算（w で割る）
NDC:          (0.08, 0.04, 0.16) ← -1〜1の正規化座標
    ↓ ビューポート変換
Screen:       (540, 490) ← ピクセル座標
```

**URP関数との対応**:
| 変換 | URP関数 | 行列 |
|------|---------|------|
| Object → World | `TransformObjectToWorld()` | `unity_ObjectToWorld` |
| World → View | `TransformWorldToView()` | `UNITY_MATRIX_V` |
| World → HClip | `TransformWorldToHClip()` | `UNITY_MATRIX_VP` |
| Object → HClip | `TransformObjectToHClip()` | `UNITY_MATRIX_MVP` |
| 法線 Object → World | `TransformObjectToWorldNormal()` | 逆転置行列 |

### ライティングモデル

**Lambert拡散反射**:
```
diffuse = max(0, dot(N, L)) × lightColor × albedo

N: 表面の法線ベクトル（正規化済み）
L: ライトの方向ベクトル（正規化済み）
→ N と L の内積 = cos(θ)
→ 光が正面なら1、平行なら0、裏なら負（clampで0に）
```

**Blinn-Phong鏡面反射**:
```
H = normalize(L + V)     ← ハーフベクトル
specular = pow(max(0, dot(N, H)), shininess) × lightColor

V: 視線方向（カメラへの方向）
H: ライト方向と視線方向の中間
→ Hが法線に近いほど鏡面反射が強い
→ shininess が大きいほどハイライトが小さくシャープに
```

**PBR（物理ベースレンダリング）**:
Phase 4以降の高度な内容。質問されたら概要のみ:
- エネルギー保存則を守るライティングモデル
- Metallic-Roughness ワークフロー
- URPのLitシェーダーが内部で使用している

### ベクトル演算

**内積（dot product）**:
```
dot(A, B) = Ax×Bx + Ay×By + Az×Bz = |A|×|B|×cos(θ)

用途:
- ライティング（N·L）: 面への光の当たり具合
- フレネル（N·V）: 視線と面の角度
- 投影: あるベクトルを別の方向に射影
```

**外積（cross product）**:
```
cross(A, B) = 両方に垂直なベクトル

用途:
- 法線計算: 2つの辺ベクトルから面の法線を求める
- TBN行列の構築: Tangent Space ↔ World Space変換
```

### テクスチャとUV

**UV座標の意味**:
```
(0,0) -------- (1,0)
  |              |
  |   テクスチャ  |
  |              |
(0,1) -------- (1,1)

UV = (0.5, 0.5) → テクスチャの中央
UV > 1 または < 0 → Wrap Mode (Repeat/Clamp) に依存
```

**ミップマップ**:
```
Level 0: 1024×1024（元解像度）
Level 1:  512×512
Level 2:  256×256
...
→ カメラから遠いほど低解像度のミップを使用
→ メモリは約1.33倍使うが、エイリアシングとGPU帯域を削減
```

### レンダリングパイプライン

**GPUパイプラインの概要**:
```
[頂点データ]
    ↓
[頂点シェーダー] ← 頂点ごとに実行。座標変換、法線変換
    ↓
[ラスタライズ] ← 三角形をピクセルに分解。Varyingsを補間
    ↓
[フラグメントシェーダー] ← ピクセルごとに実行。色を計算
    ↓
[出力マージ] ← 深度テスト、ブレンド、ステンシル
    ↓
[フレームバッファ] → 画面表示
```

**重要な概念**:
- 頂点シェーダーは頂点数分だけ実行される（数百〜数万回）
- フラグメントシェーダーはピクセル数分実行される（数百万回）
- → フラグメントシェーダーの負荷がパフォーマンスに直結

### 深度バッファとステンシルバッファ

**深度バッファ（Zバッファ）**:
```
各ピクセルにカメラからの距離を記録
→ 新しいピクセルの深度が既存値より奥なら描画スキップ
→ これにより前後関係を正しく表現

ZWrite On:  深度バッファに書き込む（不透明オブジェクト）
ZWrite Off: 書き込まない（透過オブジェクト）
ZTest LEqual: 既存値以下なら描画（デフォルト）
```

**ステンシルバッファ**: Phase 3で詳しく。
マスク処理（ポータル、アウトライン、特殊エフェクト）に使う整数バッファ。

### ブレンディング（アルファブレンド）

**Blend命令の意味**:
```hlsl
Blend SrcAlpha OneMinusSrcAlpha
// 最終色 = シェーダー出力色 × SrcAlpha + 既存色 × (1 - SrcAlpha)

// アルファ0.5の赤を白い背景に描画:
// = (1,0,0) × 0.5 + (1,1,1) × 0.5
// = (0.5, 0, 0) + (0.5, 0.5, 0.5)
// = (1.0, 0.5, 0.5) ← ピンク色
```

**よく使うBlendモード**:
| Blend命令 | 効果 |
|-----------|------|
| `Blend SrcAlpha OneMinusSrcAlpha` | 標準半透明 |
| `Blend One One` | 加算合成（光、パーティクル） |
| `Blend DstColor Zero` | 乗算合成（影、汚れ） |
| `Blend One OneMinusSrcAlpha` | プリマルチプライドアルファ |

## 数学的な説明のルール

- 数式はKaTeX記法で記述する
- 行列は具体的な4×4の数値例を1つ以上示す
- ベクトル演算は2D→3Dの順で説明する
- 「直感的にはこういうこと」という説明を数式の前に入れる
- 三角関数が出てきたら角度と値のペアを示す

### KaTeX例

インライン: $\cos(\theta) = \frac{A \cdot B}{|A| \cdot |B|}$

ブロック:
$$
\text{diffuse} = \max(0, \hat{N} \cdot \hat{L}) \times C_{\text{light}} \times C_{\text{albedo}}
$$
